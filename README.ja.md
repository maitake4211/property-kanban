# Property Kanban

[English README is here](README.md)

ノートのfrontmatterプロパティを唯一のデータソースとする [Obsidian](https://obsidian.md) 用Kanbanタスクボードプラグインです。
ボード上でのカードの作成・移動・削除がノートのプロパティに即座に反映され、ノートを編集すればボードにも反映されます。

## 機能

### Kanbanボード

- 任意のfrontmatterフィールドでカードを自動グルーピング
- カードのドラッグ&ドロップで列間を移動 → frontmatterが自動更新
- カードタイトルをクリックでノートを開く（すでに別タブで開いているノートは新規タブを作らず、そのタブに移動）
- ファイルの変更・作成・削除をリアルタイムに検知してボードを自動更新

### スイムレーン表示

2つのプロパティで二次元のグルーピングが可能です。

- **レーン**: 大枠のグループ（例: 案件）→ 横に並ぶ行
- **カラム**: レーン内の分類（例: ステータス）→ 行内の列

```
┌─ 案件A ──────────────────────────┐
│ | 未着手 | 進行中 | 完了 |        │
│ | card   | card   | card |        │
└──────────────────────────────────┘
┌─ 案件B ──────────────────────────┐
│ | 未着手 | 進行中 | 完了 |        │
│ | card   | card   | card |        │
└──────────────────────────────────┘
```

カードはレーン間もドラッグ&ドロップで移動でき、レーンとカラム両方のfrontmatterが同時に更新されます。

### カード作成

- ボード上部の「+ 新規タスク」ボタン、または各列の「+ カード追加」から作成
- 既存タスクのfrontmatterから自動検出した値がサジェスト候補として表示される
- 候補にない値もその場で自由入力でき、新しいプロパティ値をそのまま設定できる
- 作成と同時に指定フォルダにfrontmatter付きのMarkdownノートが生成される

### カード削除

- カードにホバーすると右上に `⋯` メニューボタンが表示される
- メニューから「削除」を選択するとノートをゴミ箱に移動
- 確認ダイアログ付き（「次回から確認しない」チェックでスキップ可能）
- 削除時は、その子ノートの親リンクを解除し、親ノートの子リストからも自動で除外される

### クイックアクションボタン

ワンクリックで特定のfrontmatterフィールドを指定値に書き換えるボタンをカード上に表示できます（例: ステータスを「完了」にする）。

- 設定画面の「クイックアクションボタン」で有効化し、ラベル・対象フィールド・設定する値を指定
- すでに対象値になっているカードにはボタンを表示しない
- クリックすると対象ノートのfrontmatterを即座に更新し、Notice で通知

### 親子関係

カード同士の親子関係を frontmatter の wikilink で管理できます。Obsidian のグラフビューやバックリンクで階層を追えるようになります。

- **データの持ち方**: 子ノートの親リンク（例: `parent: "[[親タスク]]"`）が正データ。親ノートの子リスト（wikilink のリスト）はプラグインが自動生成する派生データ
- **親を設定**: カードの `⋯` メニュー →「親を設定…」でタスクを検索して選択（自分と子孫は候補から除外され、循環を防止）
- **親を解除**: `⋯` メニュー →「親を解除」
- **子を追加**: 親カードの `⋯` メニュー →「子を追加…」で、親が設定済みの状態で新規カード作成モーダルが開く
- **多階層対応**: 親をたどる構造なので孫・ひ孫も表現可能
- **再構築**: 手編集などで崩れた場合、設定画面の「親子リンクを再構築」またはコマンドパレットから子リストを一括再生成

### ビュー設定（⚙）

ヘッダーの歯車アイコンをクリックすると展開されるポップオーバーで以下の操作が可能です。

| 機能 | 説明 |
|------|------|
| レーン / カラム | グルーピングに使うfrontmatterフィールドを切り替え |
| ズーム | ボードの拡大/縮小（50%〜200%） |
| カードに表示する項目 | カード上に表示するプロパティの表示/非表示 |
| 列の並び替え・表示 | レーン・カラムの順序変更（D&D）と表示/非表示の切り替え |

### カード表示プロパティ

カード上にタグとして表示するfrontmatterフィールドを自由に設定できます。

- 任意のフィールドを追加・削除可能
- フィールドごとに色（blue, green, red, yellow, purple, gray）を選択
- 設定画面またはビュー設定ポップオーバーから切り替え

## インストール

### コミュニティプラグインから（推奨）

1. 設定 → コミュニティプラグイン → 閲覧 を開く
2. 「Property Kanban」を検索
3. インストールして有効化

### 手動インストール

1. 最新の [Release](https://github.com/maitake4211/property-kanban/releases) から `main.js`, `manifest.json`, `styles.css` をダウンロード
2. Vault内に `.obsidian/plugins/property-kanban/` フォルダを作成
3. ダウンロードした3ファイルをそのフォルダにコピー
4. Obsidianを再起動し、設定 → コミュニティプラグイン → 「Property Kanban」を有効化

## 使い方

1. プラグインを有効化する
2. 左サイドバーのダッシュボードアイコンをクリック、またはコマンドパレットで `Property Kanban: ボードを開く` を実行
3. ボードが開く

## 設定項目

| 設定 | 説明 | デフォルト値 |
|------|------|-------------|
| タスクフォルダ | タスクノートの保存先フォルダパス | `tasks` |
| カード表示プロパティ | カード上に表示するfrontmatterフィールドの一覧と色 | category, project, due, created |
| クイックアクション: ボタンを表示 | カード上にクイックアクションボタンを出すか | OFF |
| クイックアクション: ボタンのラベル | カードに表示する文字 | `Done` |
| クイックアクション: 対象フィールド | 書き換える frontmatter フィールド名 | `status` |
| クイックアクション: 設定する値 | ボタン押下時にセットされる値 | `Done` |
| 親フィールド名 | 子ノートに書き込む親リンクのフィールド名 | `parent` |
| 子フィールド名 | 親ノートに自動生成する子リストのフィールド名 | `children` |
| 子リストを自動生成 | 親ノートに子の wikilink リストを維持するか（OFFで親リンクのみ） | ON |
| 新規カードに空の親欄を出す | 親未指定でも空の親フィールドを書き込むか | OFF |

グルーピング（レーン/カラム）、並び順、表示/非表示、ズームの設定はボード上のビュー設定から行い、自動保存されます。

UIは英語がデフォルトで、Obsidianの言語設定が日本語の場合は日本語表示になります。

## 技術仕様

- frontmatterの読み書きはファイル内容を直接パースして行う（metadataCacheに依存しない）
- グルーピングのフィールドとカラム値はタスクファイルのfrontmatterから自動検出
- ボードの更新はデバウンス処理（150ms）で連続イベントを集約
- カード削除は `vault.trash()` を使用し、ゴミ箱に移動（完全削除ではない）
- データを外部に送信することは一切ありません（ネットワークリクエストなし）
